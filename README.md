# 客诉洞察

客诉数据分析平台，上传 Excel 自动完成归因分析和可视化。

---

## 背景

产品团队在处理用户客诉时，长期面临几个痛点：

1. **客诉量大，人工整理效率低。** 每周数千条客诉反馈，产品需要逐条阅读、手动分类、整理到 Excel 里，耗费大量时间在重复性工作上，真正用于分析和决策的时间反而很少。

2. **归因粗、不准确。** 现有的客诉分类体系比较粗（如"播放异常"、"音效相关问题"），同一个大类下可能包含十几种完全不同的具体问题。产品看到"播放异常占比40%"，但不知道到底是卡顿、中断、没声音还是格式不支持，无法精准定位问题和推动解决。

3. **缺乏趋势感知。** 某类问题是在恶化还是改善？某个版本发布后客诉有没有上升？这些问题靠手动统计很难持续跟踪。

4. **规则维护成本高。** 即使建立了归因规则，也是写在文档或脑子里，换个人就不知道怎么分类了，缺乏一个统一的、可维护的规则体系。

**解决思路：** 做一个轻量化的本地工具，让产品团队上传 Excel 就能自动完成归因分析和可视化。通过关键词规则做细粒度归因，精确匹配 + jieba 分词匹配双重机制提高召回率；对未归因客诉自动提取高频词辅助补充规则；产品可在页面上直接维护规则，不依赖开发。所有数据在本地处理，不上传到任何外部服务器。

**不用大模型的原因：** API 调用有成本，且客诉数据涉及用户隐私不适合传到外部。关键词规则方案虽然没有 AI 那么"智能"，但透明可控、零成本、产品自己能维护，对当前阶段的需求来说是最务实的选择。

---

## 功能

- **字段映射** — 适配任意格式的 Excel，上传后手动选择对应列
- **关键词归因** — 在已有粗分类基础上，自动做细粒度归因（如"音效相关问题"→"均衡器无效"）
- **Dashboard 看板** — 分类分布饼图、客诉趋势折线图、机型 TOP10、词云、TF-IDF 关键词
- **待归因分析** — 自动提取未归因客诉的高频词，辅助补充规则
- **明细表** — 按时间/分类/机型等关键词搜索筛选，支持导出 Excel
- **规则管理** — 在页面上新增/删除归因规则，无需改代码
- **数据安全** — 所有数据在本地处理，不上传到任何服务器

## 安装和启动

### 方式一：双击启动（Windows）

1. 确保电脑已安装 Python（[下载地址](https://www.python.org/downloads/)，安装时勾选 "Add Python to PATH"）
2. 双击 `启动.bat`
3. 首次启动会自动安装依赖（需要联网，约 2-3 分钟）
4. 安装完成后浏览器自动打开 `http://127.0.0.1:5000`

### 方式二：命令行启动

Windows CMD：
```
D:
cd "\年后\用户增长\客诉"
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
python app.py
```

浏览器打开 `http://127.0.0.1:5000`。

## 使用流程

### 1. 上传 Excel

打开页面后，点击上传区域选择 Excel 文件，或直接拖拽文件到上传区。

支持 `.xlsx` 和 `.xls` 格式。

### 2. 字段映射

上传后页面会列出 Excel 的所有列名，通过下拉框告诉系统每一列是什么：

| 字段 | 是否必选 | 说明 |
|------|---------|------|
| 时间 | 必选 | 客诉发生的时间，用于趋势分析 |
| 客诉内容 | 必选 | 用户反馈的文字，用于分词和归因 |
| 机型/型号 | 可选 | 映射后会生成机型 TOP10 排行 |
| 已有分类 | 可选 | 映射后会自动做细分归因和待归因分析 |
| 客服反馈 | 可选 | 映射后会和客诉内容合并进行文本分析 |

选好后点击「开始分析」。

### 3. 查看 Dashboard

分析完成后自动跳转到 Dashboard，包含：

- **指标卡片** — 总客诉数、已归因数、待归因数、TOP 问题
- **问题分类分布** — 饼图，可切换粗分类/细分归因
- **机型 TOP 10** — 哪些机型投诉最多
- **客诉趋势** — 按日/周/月查看数量变化
- **词云** — 客诉中出现最多的词
- **TF-IDF 关键词** — 对客诉文本最重要的关键词
- **各分类关键词** — 每个问题分类下的代表性关键词
- **待归因分析** — 未被规则匹配的客诉的高频词和样例（仅映射了「已有分类」时显示）

### 4. 查看明细表

切换到「明细表」标签页：

- 顶部搜索框可按关键词搜索
- 点击「导出 Excel」下载带细分归因的完整数据

### 5. 管理归因规则

切换到「规则管理」标签页：

- 查看当前所有规则
- 点击规则右侧的 ✕ 删除
- 底部表单添加新规则：填写粗分类、细分归因、关键词（逗号分隔）
- 修改规则后，重新上传 Excel 即可生效

**补充规则的技巧：** Dashboard 底部的「待归因分析」板块会列出未归因客诉的高频词，点击词语会自动填入规则管理的关键词输入框，方便快速补充。

## 归因规则说明

规则存储在 `config/rules.json`，按粗分类组织：

```json
{
  "音效相关问题": [
    {"sub_category": "均衡器无效", "keywords": ["均衡器", "EQ", "调节无效"]},
    {"sub_category": "音质差", "keywords": ["杂音", "破音", "噪音"]}
  ]
}
```

当前预置了 14 大类、40+ 细分归因规则，基于音乐反馈月报的分类体系，覆盖播放、下载、客户端、广告、会员、账号、曲库、歌词、歌单、本地音乐、设置功能、新功能需求、投诉、高端机等场景。**该规则仅基于个人认知预设，产品团队可根据实际业务调整：**

- 在页面「规则管理」标签页直接新增/删除
- 或手动编辑 `config/rules.json` 文件

匹配逻辑：
1. 扫描客诉内容 + 客服反馈文本
2. 精确匹配关键词（权重高）+ jieba 分词后匹配（权重低），提高召回率
3. 按命中权重取最佳匹配，未命中标记为"其他/待归因"

规则可在页面上维护，也可直接编辑 `config/rules.json` 文件。

---

## 停用词

`config/stopwords.txt` 存放分词时需要过滤的词（如"的"、"了"、"是"等）。

每行一个词，修改后不需要重启，重新上传 Excel 即可生效。

## 常见问题

**Q：启动后浏览器没有自动打开？**
A：手动在浏览器输入 `http://127.0.0.1:5000`。

**Q：安装依赖很慢或失败？**
A：使用清华镜像：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`

**Q：归因不准怎么办？**
A：去「规则管理」调整关键词。Dashboard 底部的「待归因分析」会告诉你哪些词被遗漏了。

**Q：改了停用词/规则没生效？**
A：停用词修改后重新上传 Excel 即可。规则修改后也需要重新上传。

**Q：数据会上传到外部吗？**
A：不会，所有处理都在本地完成。
